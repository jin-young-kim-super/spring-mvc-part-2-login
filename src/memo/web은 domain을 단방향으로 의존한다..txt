Domain 패키지와 Web 패키지가 분리돼 있다 
-> 일단, 도메인은 비지니스 핵심 모델(비지니스 규칙, 비지니스 행동)이다. 그러나 Web 분야는 화면, UI, 기술 인프라 등등의 영역이다. 
고로, 사용하는 기술이 바뀌어도 도메인은 절대 바뀌지 않으므로 분리하는 것이 좋다. 
그리고 도메인은 web이 바뀌어도 절대 변경되면 안 되니깐 Web은 도메인을 참조해도, 도메인은 절대 Web을 참조하지 않드로고 Web->도메인으로 단방향으로 의존 관계를 맺어야 한다. 
그래서 web 패키지를 다 지워우거나 전체를 다 갈아 엎어거나, 뷰 컨트롤러를 전부 API 컨트롤러로 바꾸든 오만짓을 하더라도
도메인 모델에 컴파일 에러가 나게 설계하지 않느 것이 중요하다.
